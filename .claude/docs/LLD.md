- **Interfaces**: Each function takes filter objects (dates, SGA/SGM filters, advanced filters) and returns typed arrays. Return types defined in `src/types/`.
- **Consumers**: API routes under `src/app/api/` call these functions directly.
- **Patterns**:
  - All BigQuery queries use `@paramName` parameterized syntax via `bigquery.query({ query, params })`
  - Raw results mapped through transform functions using `toString()`, `toNumber()` from `src/types/bigquery-raw.ts`

## Cross-Cutting Patterns

### Date Handling
- `src/lib/utils/date-helpers.ts`: `buildDateRangeFromFilters()` for converting UI filters to SQL date ranges, `formatCurrency()` / `formatAumCompact()` for display
- `src/lib/utils/format-helpers.ts`: `formatDate()` / `formatDateTime()` for display formatting
- `src/lib/utils/sga-hub-helpers.ts`: Week/quarter date math (Monday-anchored weeks, quarter start/end)
- BigQuery queries use `DATE()` or `TIMESTAMP()` wrappers depending on the field — see `bq-patterns.md` for per-field guidance

### NULL / Default Handling
- BigQuery: COALESCE with sensible defaults in SELECT clauses — never filter NULLs (keeps records)
- TypeScript: `toNumber()` and `toString()` from `src/types/bigquery-raw.ts` handle BigQuery result coercion
- Nullable fields in types use `| null` — components handle null display

### Export / CSV
Two distinct export paths:
1. **ExportButton** (`src/components/ui/ExportButton.tsx`): Uses `Object.keys()` on data — auto-includes new fields. No manual update needed.
2. **ExportMenu** (`src/components/dashboard/ExportMenu.tsx`) + **MetricDrillDownModal** (`src/components/sga-hub/MetricDrillDownModal.tsx`): Explicit column mappings — **must be updated manually** when types change.
3. **Google Sheets export** (`src/lib/sheets/google-sheets-exporter.ts`): Multi-tab formatted spreadsheets with conditional formatting.

### Error Handling
- API routes: try/catch wrapping query calls, return `NextResponse.json({ error: message }, { status: 500 })`
- Prisma: `retryDatabaseOperation()` wrapper in `src/lib/users.ts` for Neon connection timeouts
- Components: `ErrorBoundary` wraps the entire dashboard layout; feature-specific error boundaries via `src/components/ui/DashboardErrorBoundaries.tsx`

### Import Conventions
- Path alias: `@/*` maps to `./src/*` (from `tsconfig.json`)
- Barrel files exist at: `src/components/ui/index.ts`, `src/components/advisor-map/index.ts`, `src/components/games/pipeline-catcher/index.ts`, `src/lib/semantic-layer/index.ts`
- No barrel file in `src/lib/queries/` — each file imported directly by path
- Merge rule: Never add a second import from the same module — merge into the existing import statement

## Interface Map

### BigQuery → TypeScript
```
BigQuery query result (raw rows) → transform via toString()/toNumber() → Typed interface (e.g., DetailRecord)
```
Defined in `src/types/bigquery-raw.ts` (raw types) → `src/types/dashboard.ts` (clean types)

### Query Function → API Route
```
Query function (src/lib/queries/*.ts) returns typed array → API route returns NextResponse.json(data)
```
Most API routes are thin pass-throughs: parse request body/params → call query function → return JSON.

### API Route → Component
```
Component fetches from /api/... → receives JSON → renders via Tremor/Recharts components
```
Components use `fetch()` or `src/lib/api-client.ts` to call API routes.

### Prisma Model → API Response
```
Prisma query (e.g., prisma.user.findMany()) → map to safe interface → return JSON
```
User data stripped of `passwordHash` before returning. Goal data returned as-is.
