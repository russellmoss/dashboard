## System Purpose

Savvy Wealth's internal recruiting funnel analytics dashboard. Replaces Tableau for tracking financial advisor prospects from initial contact through joining as 1099 employees. ~35 internal users across 8 roles (RevOps admins, managers, SGMs, SGAs, recruiters, capital partners, viewers) use it for pipeline decisions, quota tracking, and performance coaching.

## Data Flow

### Postgres Read Path
```
Prisma client (src/lib/prisma.ts) → query/utility file → API route → component
```
Serves: user accounts, goals (weekly/quarterly), forecasts, dashboard requests, saved reports, game scores, GC Hub advisor data, MCP API keys.

### BigQuery Read Path
```
BigQuery SDK (src/lib/bigquery.ts) → query function (src/lib/queries/) → API route → component
```
Serves: recruiting funnel analytics, pipeline metrics, conversion rates, SGA/SGM leaderboards, activity data, advisor locations.

### Key constraint: No API route reads from both Postgres and BigQuery in the same request, with three exceptions:
- `admin-quarterly-progress.ts` — joins BQ funnel counts with Prisma quarterly goals
- `sgm-quota.ts` — joins BQ pipeline data with Prisma SGM quota goals
- `advisor-locations.ts` — joins BQ location data with Prisma address overrides

### Write Paths
- **Salesforce → BigQuery**: Data Transfer Service (6 jobs, every 6 hours, incremental)
- **Google Sheets → Prisma**: GC Hub revenue sync (daily cron at 6:30 AM UTC)
- **User actions → Prisma**: Goal edits, request submissions, game scores, forecast scenarios, saved reports
- **Dashboard → Google Sheets**: Export feature writes formatted spreadsheets
- **Dashboard → Wrike**: Request submissions sync to Wrike tasks (background)
- **Dashboard → SendGrid**: Password reset emails

## Authentication and Authorization

### Auth Stack
- **Provider**: NextAuth.js v4 with two providers:
  - `CredentialsProvider` — email/password (bcrypt hashed, stored in Prisma `User`)
  - `GoogleProvider` — restricted to `@savvywealth.com` domain, requires pre-provisioned User record
- **Session strategy**: JWT (24-hour maxAge). No database sessions. No `PrismaAdapter` in use.
- **Middleware**: `src/middleware.ts` — protects `/dashboard/*` and `/api/*` routes. Redirects unauthenticated users to `/login`. Enforces role-based route restrictions for `recruiter` and `capital_partner` roles.

### Role System (8 roles, defined in `src/lib/permissions.ts`)
| Role | Pages | Key Capabilities |
|---|---|---|
| `revops_admin` | All 19 pages | Export, manage users, manage requests, run scenarios |
| `admin` | 17 pages | Export, manage users, run scenarios |
| `manager` | 11 pages | Export |
| `sgm` | 7 pages | Export, auto-scoped to own data |
| `sga` | 8 pages | Export, auto-scoped to own data |
| `viewer` | 6 pages | Read-only |
| `recruiter` | 2 pages (Recruiter Hub, Settings) | Export, scoped to own agency |
| `capital_partner` | 2 pages (GC Hub, Settings) | Export, scoped to own agency |

### Permission Enforcement (3 layers)
1. **Middleware** (`src/middleware.ts`): Blocks recruiter/capital_partner from unauthorized routes (API and page)
2. **Dashboard layout** (`src/app/dashboard/layout.tsx`): Client-side redirect for restricted roles
3. **API routes**: `getServerSession` → `getSessionPermissions` → role/page checks per route. SGA/SGM users auto-scoped via `sgaFilter`/`sgmFilter`.
