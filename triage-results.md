## Bucket 2 — Need Human Input (5 questions surfaced via AskUserQuestion)

These were merged + de-duplicated from 11 candidate questions. Many were closely related (e.g., audit-toggle layout + citation-pill clutter + draft preservation are all "UX trade-offs"; comment visibility + citation persistence + refinement SLA are "product semantics"). I bundled where possible.

### Q1. Citation persistence on canonical edits (C6 + Gemini Q1) ⭐ HIGHEST IMPACT
When a manager edits a narrative/strength/weakness, what happens to citations?
- **A — Accept loss:** edits strip citations on the canonical row; audit toggle is the historical lens. Simplest; matches current schema.
- **B — Coordinate sales-coaching change:** extend `EditEvaluationRequest` with optional `narrative_citations: Citation[]` etc. Preserves citations through edits but blocks 5b-1-UI on a backend change.
- **C — Display-only preservation:** UI keeps citations visually but server discards them on save (i.e., next reload removes them). Worst of both worlds — flag and avoid.

### Q2. Comment visibility for reps (C7)
Who can READ utterance-level comments via GET `/transcript-comments`?
- **A — Managers + admins only.** Reps blocked entirely. Strictest; safest default.
- **B — Reps see comments only after eval reveal.** Couples to reveal_policy; aligns with existing reveal flow.
- **C — Reps see all comments immediately.** Most transparent; risks exposing private coaching framing.

### Q3. Authority-Lost UX + draft preservation (C9 + Gemini I1)
On authority-lost (snapshot reassignment), what's the right UX?
- **A — Banner + "Return to queue" button** (user controls when to leave; drafts preserved as long as user stays).
- **B — Auto-redirect after 1.5s** (matches the original spec; loses drafts).
- **C — Banner + button + localStorage draft auto-save** so a reload restores typed text. (Most defensive; biggest scope.)

### Q4. CI schema mirror source for acceptance test `(m)` (C8)
Where does `check:schema-mirror` get the upstream schema in CI?
- **A — Skip in CI; mark `(m)` "verified locally only".** Pragmatic; doesn't satisfy spec literally.
- **B — Vendor a snapshot file in this repo,** manually re-synced when sales-coaching changes. Adds ops surface.
- **C — Fetch from sales-coaching's GitHub raw URL** in a CI step (token-gated for private repo).
- **D — Publish sales-coaching's schemas as an npm package** and import + diff. Cleanest long-term; biggest scope.

### Q5. Audit toggle layout + citation clutter (Gemini Q2 + Q1) ⭐ BUNDLED UX
Two related UX trade-offs that should be answered together:
- (a) **Audit toggle:** global side-by-side comparison (matches spec, can overwhelm) vs per-field "AI Original" popover (less cognitive load, deviates from spec).
- (b) **Citation pills:** render every citation as separate pill (faithful to data) vs group same-type into one pill (`[💬 47, 48, 49]`) vs dedupe by target.
- **A — Spec-faithful:** global toggle + every citation as a pill (default).
- **B — UX-optimized:** per-field popover + grouped pills. Better for managers reviewing dozens of evals; deviates from spec.
